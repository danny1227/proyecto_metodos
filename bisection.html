Descripcion:

En matemáticas, el método de bisección es una técnica sencilla para encontrar soluciones numéricas de una ecuación con una incógnita. Entre todos los métodos numéricos, el método de bisección es el más sencillo para resolver la ecuación trascendental.


Metodo de bisección

Definición :

El método de bisección se utiliza para encontrar las raíces de una ecuación polinómica. Se separa el intervalo y se subdivide el intervalo en el que se encuentra la raíz de la ecuación. El principio de este método es el teorema intermedio para funciones continuas. Funciona reduciendo la brecha entre los intervalos positivo y negativo hasta que se acerca a la respuesta correcta. Este método reduce la diferencia tomando la media de los intervalos positivo y negativo. Es un método sencillo y relativamente lento.
El método de bisección también se conoce como método de reducción a la mitad del intervalo, método de búsqueda de la raíz, método de búsqueda binaria o método de dicotomía. Consideremos una función continua "f" que está definida en el intervalo cerrado [a, b], está dada con f(a) y f(b) de distinto signo. Entonces, por el teorema intermedio, existe un punto x que pertenece a (a, b) para el que f(x) =0.

Mi experiencia con el metodo

En cualquier software en donde queramos aplicarlo se hace muy sencillo el codificarlo, ya sea por medio de fórmulas en Excel o usando lenguajes de programación como los con c++ o Matlab. La codificación del algoritmo es muy entendible, y resulta simple de comprender como funciona.

Origen

Aunque hay pocos conocimientos concretos sobre el desarrollo del método de bisección, podemos deducir que se desarrolló poco después de que Bernard Bolzano demostrara por primera vez el Teorema del Valor Intermedio en 1817 (Edwards 1979). Parece que se utilizó como prueba de un teorema intermedio a la prueba general que Bolzano estaba desarrollando para el Teorema del Valor Intermedio.

-Edwards, C. H. (1979). Bolzano, Cauchy, and Continuity. The Historical Development of the Calculus (pp. 308, 309). New York, NY: Springer New York


ALGORITMO

ENTRADA:
 Intervalo [a, b]; Tolerancia TOL;
Cantidad máxima de iteraciones IT;
HACER i = 1 // i es la variable que controla la cantidad de iteraciones
FA = f(a)
MIENTRAS i ≤ IT REPETIR
HACER p = (a+b)/2 // (calculando pi)
HACER FP = f(p)
SI FP == 0 o |(b-a)/2| < TOL ENTONCES
  SALIDA (p) // procedimiento finalizado satisfactoriamente
  PARAR
HACER i=i + 1 // control de cantidad de iteraciones
SI FA * FP> 0 ENTONCES
  HACER a = p // calcular ai, bi
  HACER FA = FP
SI NO
  HACER b = p
  FIN MIENTRAS
SALIDA (“El método fracasó después de “ + IT + “ iteraciones”)
PARAR

Ventajas:

* Siempre es convergente
* El aumento del número de iteraciones siempre produce una raíz más precisa, es decir el error se puede controlar.
* No requiere cálculos complejos. Para realizar el método de bisección, todo lo que necesitamos es calcular el promedio de dos números.
* Existe un límite de error garantizado, que disminuye con cada iteración sucesiva. El límite de error disminuye en ½ con cada iteración.
* Es simple y fácil de programar en computadora.
* Es rápido en caso de multiplices raices.

Desventajas :

* No se puede encontrar la raíz de algunas ecuaciones. Por ejemplo: f (x) = x^2 ya que no hay valores entre corchetes.
* No se puede aplicar en un intervalo donde la función toma valores del mismo signo.
* No determina raices complejas.
* Si bien la convergenia está asegurada, está puede llevar mucho más tiempo que otros metodos.
* No se puede aplicar si hay discontinuidades en el intervalo, debe ser función lineal.
